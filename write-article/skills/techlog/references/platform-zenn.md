# Zenn プラットフォームガイド

## Front Matter（必須）

```yaml
---
title: "記事タイトル"
emoji: "🚀"
type: "tech"
topics: ["react", "nextjs", "typescript"]
published: true
---
```

| フィールド | 説明 |
|---|---|
| `title` | 記事タイトル（必須） |
| `emoji` | アイキャッチ絵文字（1文字、記事内容に関連するものを選ぶ） |
| `type` | `"tech"`（技術記事）or `"idea"`（アイデア記事）。技術ブログなら常に `"tech"` |
| `topics` | タグ配列。最大5つ。主要技術を英小文字で指定 |
| `published` | `true` で公開、`false` で下書き。常に `true` で出力 |

### emoji選定のコツ

記事内容に合った絵文字を選ぶ。迷ったらこれ:
- ツール/ライブラリ紹介: `"🔧"` `"⚡"` `"🛠"`
- チュートリアル: `"📝"` `"📚"` `"🎓"`
- パフォーマンス/最適化: `"🚀"` `"⚡"` `"🏎"`
- エラー/デバッグ: `"🐛"` `"🔍"` `"💥"`
- 比較/選定: `"⚖"` `"🤔"` `"📊"`
- セキュリティ: `"🔒"` `"🛡"` `"🔑"`

### topics選定のコツ

- ソース素材で使われている技術名をそのまま入れる
- 一般的な表記を使う（`"nextjs"` not `"next.js"`、`"typescript"` not `"ts"`）
- 広すぎるタグ（`"programming"`）は避け、具体的に

## Zenn独自のMarkdown拡張

### メッセージボックス

```markdown
:::message
これは通常のメッセージです。補足情報や注意点に使う。
:::

:::message alert
これは警告メッセージです。破壊的変更や重要な注意に使う。
:::
```

### アコーディオン（詳細折りたたみ）

```markdown
:::details タイトル
折りたたまれる内容。長いコードや補足情報に使う。
:::
```

### 数式（KaTeX）

```markdown
$$
e^{i\pi} + 1 = 0
$$
```

### ファイル名付きコードブロック

````markdown
```ts:src/app/page.tsx
export default function Home() {
  return <h1>Hello</h1>
}
```
````

ファイル名を付けると、読者がどのファイルを編集すべきかが一目でわかる。技術記事では積極的に使う。
